# Security Testing

**Last Updated:** March 14, 2026

SveltyCMS implements rigorous security testing to ensure the highest level of protection. All security tests follow the **no-backdoor policy** - tests must pass through real security guards.

| Key Derivation (Argon2id) | 6     | ✅ Passing |

**Test Files:**

- [`tests/unit/utils/crypto.test.ts`](../../tests/unit/utils/crypto.test.ts)
- [`tests/unit/utils/preview-verification.test.bun.ts`](../../tests/unit/utils/preview-verification.test.bun.ts)

### 2. Preview Security (Enterprise Handshake)

We verify the integrity of the Live Preview handshake to prevent unauthorized access to draft content.

**Coverage:**

- ✅ **Signed Tokens**: Verification of HMAC-SHA256 signatures using `PREVIEW_SECRET`.
- ✅ **Temporal Expiry**: Hard rejection of tokens older than the 30-minute window.
- ✅ **Identity Binding**: Ensuring tokens are bound to specific `entryId` and `userId`.
- ✅ **Handshake Origin**: Validation of `postMessage` origins in the preview iframe.

## Running Security Tests

```bash
# Run cryptography and preview verification tests
bun test tests/unit/utils/crypto.test.ts
bun test tests/unit/utils/preview-verification.test.bun.ts

# Run auth security tests
npm run test:unit -- auth-lockout
```
