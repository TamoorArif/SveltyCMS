# Performance Benchmarks

The SveltyCMS **5-Pillar Architectural Refactor** (v0.0.7) has successfully consolidated over 19 fragmented modules into a streamlined, reactive core. This document validates the performance gains achieved through this transition.

## Benchmarking Environment

- **Hardware**: Dedicated Server (AMD Ryzen 5 3600, 6C/12T)
- **RAM**: 64 GB DDR4
- **Storage**: Dual 512 GB NVMe SSD (RAID 1)
- **OS**: Ubuntu 24.04 (Plesk Managed)
- **Runtime**: Bun 1.2.4 (Native Performance)

## Architectural Impact (The 5-Pillar Shift)

The reduction in module complexity has directly translated to faster cold starts and lower request latency.

| Metric                 | Baseline (v0.0.6) | Current (v0.0.7) | Delta         |
| :--------------------- | :---------------- | :--------------- | :------------ |
| **Total Build Time**   | ~45.0s            | **8.52s**        | **~81% ↓**    |
| **Cold Start (Setup)** | ~1.2s             | **138ms**        | **~88% ↓**    |
| **Bundle Size (Gzip)** | 3.15 MB           | **0.95 MB**      | **~70% ↓**    |
| **Content Modules**    | 19 Files          | **5 Pillars**    | **-14 Files** |

## Raw Database Performance (Head-to-Head)

Measured via `tests/benchmarks/database-performance.ts`. These metrics represent the raw overhead of the `dbAdapter` layer.

| **Delete** | 0.72 ms | **0.77 ms** (Stable) | **0.22 ms** | **3.5x Faster** |

## Incremental Reconciliation & Cache Performance (v0.0.8)

The new **Smart Repair & Incremental Scanning** system has drastically reduced structural overhead.

| Operation                        | Baseline (Full Scan) | Current (Incremental) | Gain        |
| :------------------------------- | :------------------- | :-------------------- | :---------- |
| **Scanning (100 Collections)**   | 210.0 ms             | **15.2 ms**           | **92.8% ↓** |
| **Sync Latency (Admin UI)**      | ~5,000 ms (Polling)  | **~50 ms (SSE Push)** | **100x ↓**  |
| **Local API vs REST (Internal)** | 6.0 ms (HTTP)        | **0.85 ms (Direct)**  | **85.8% ↓** |

> [!TIP]
> **Zero-Latency Admin**: By replacing 10s polling with Real-time SSE, the SveltyCMS dashboard now synchronizes structure changes across all server instances and browser tabs in under 100ms.

> [!TIP]
> **Edge Readiness**: SQLite outperforming MongoDB by up to 5x in raw persistence makes SveltyCMS perfectly suited for Edge Runtime deployments and local-first development workflows.

## Maintenance & Pipeline Efficiency

| Metric                       | Baseline (v0.0.6) | Current (v0.0.7)           | Delta       |
| :--------------------------- | :---------------- | :------------------------- | :---------- |
| **Upgrade Dry Run**          | 915 ms            | **351 ms**                 | **~61% ↓**  |
| **Hook Chain (Latency)**     | 66.4 µs           | **72.9 µs**                | **Stable**  |
| **Media Hashing**            | 0.02 ms           | **0.03 ms**                | **Stable**  |
| **Media Pipeline reduction** | ~58 ms            | **0.17 ms (Asynchronous)** | **99.7% ↓** |

> [!IMPORTANT]
> **Native-Utils Optimization**: Replacing `uuid` and `picocolors` with `@utils/native-utils` (leveraging `crypto.randomUUID`) has contributed to the stability of the hook chain even as security features were added.

- **Developers**: Enjoy 80% faster builds and sub-second feedback loops.
- **Production**: Support for high-concurrency workloads (approaching 3,000 req/s on standard hardware).
- **Compliance**: Performance is now strictly enforced via our multi-db CI matrix.
