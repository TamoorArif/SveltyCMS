# API Reference

SveltyCMS provides a comprehensive programmatic surface for building custom frontends, automating workflows, and synchronizing external data.

---

## 🔐 Authentication & Security

All API endpoints are gated by the **Security Middleware Pipeline** (Rate Limiting, Firewall, and Auth).

### 1. Developer Access Tokens (Programmatic)

For long-lived scripts or server-to-server integrations, use **Bearer Tokens**.

- [🔐 **API Access Tokens**](./api-access-tokens.mdx) — Guide to generating and using tokens.
- [🛡️ **User Token Management**](./user-token-management-api.mdx) — Programmatic revocation and rotation.

### 2. Session Cookies (Web App)

Used by the Admin Studio and local frontend components.

- [👥 **User Management API**](./user-management-api.mdx) — Authentication, registration, and profile updates.
- [🔐 **2FA API**](./authentication-2fa-api.mdx) — Two-factor setup and verification.

## 📡 The Response Strategy

All REST endpoints follow a unified response shape to ensure consistent client-side handling.

### Success Format (200 OK / 201 Created)

```json
{
  "success": true,
  "data": { ... },
  "performance": { "duration": 4.5 }
}
```

### Error Format (4xx / 5xx)

```json
{
  "success": false,
  "error": "UNAUTHORIZED",
  "message": "Invalid Bearer token provided."
}
```
