# SAML 2.0 / Enterprise SSO API

SveltyCMS provides a robust SAML 2.0 integration for enterprise identity providers (IdPs) like Okta, Azure AD, and PingIdentity. This system is built on top of BoxyHQ's SAML Jackson and supports multi-tenant isolation.

## Endpoints

### 1. Configure SAML Connection

**POST** `/api/auth/saml/config`

Configures or updates a SAML connection for a specific tenant.

**Authentication**: Required (Admin role)

**Body Parameters**:

- `tenant` (string): The tenant identifier.
- `product` (string): The product identifier (usually `sveltycms`).
- `rawMetadata` (string): The XML metadata from the IdP.
- `defaultRedirectUrl` (string): The URL to redirect to after successful SSO.

**Example Request**:

```json
{
  "tenant": "my-tenant",
  "product": "sveltycms",
  "rawMetadata": "<EntityDescriptor ...>...</EntityDescriptor>",
  "defaultRedirectUrl": "https://cms.example.com/admin"
}
```

### 2. Initiate SAML Login

**GET** `/api/auth/saml/login`

Initiates the SAML SSO flow by redirecting the user to their configured Identity Provider.

**Authentication**: Public

**Query Parameters**:

- `tenant` (string): Required. The tenant identifier.
- `product` (string): Optional. Defaults to `sveltycms`.

**Example**: `GET /api/auth/saml/login?tenant=my-tenant`

### 3. Assertion Consumer Service (ACS)

**POST** `/api/auth/saml/acs`

The callback endpoint where the IdP sends the SAML assertion.

**Authentication**: Public (IdP Callback)

**Body**: Multipart/Form-Data (SAMLResponse, RelayState)

**Features**:

- **Just-In-Time (JIT) Provisioning**: Automatically creates users on first login if enabled.
- **Tenant Isolation**: Strictly enforces that users are logged into the correct tenant.
- **Security**: Validates signatures, audiences, and expirations.

## Multi-Tenant Considerations

When `MULTI_TENANT` is enabled:

- Admins can only configure SAML for their own `tenantId`.
- Login requests must include the `tenant` parameter.
- The ACS endpoint validates that the assertion belongs to the requested tenant.

## Just-In-Time (JIT) Provisioning

If `SAML_JIT_PROVISIONING` is set to `true` in system settings:

- New users will be created automatically upon successful SAML authentication.
- Default role for JIT users is `VIEWER` (configurable).
- Users are automatically associated with the authenticated tenant.
