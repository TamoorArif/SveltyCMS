# Trash & Soft Delete API

SveltyCMS implements a **Native Soft Delete** system. By default, when an entry is deleted via the API or GUI, it is moved to the **Trash** rather than being permanently removed from the database.

## 🗑️ How Soft Delete Works

### Mangle-on-Delete

To prevent **Unique Index Collisions** (e.g., trying to create a new post with the same slug as a deleted one), SveltyCMS automatically mangles unique fields upon deletion.

- **Original**: `my-awesome-post`
- **Trashed**: `my-awesome-post_DELETED_1710793389`

### Global Query Filtering

The `safeQuery` middleware automatically filters out trashed items from all standard CMS queries. Trashed items are only visible when explicitly requested via the `includeDeleted` option in the database adapter.

## 🚀 Quick Reference

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| GET | `/api/trash` | List all soft-deleted items across collections |
| POST | `/api/trash` (body: `restore` action) | Restore a soft-deleted item |
| DELETE | `/api/collections/[id]/[entryId]?permanent=true` | Permanently delete (skip trash) |

## 📡 Endpoints

### 1. List Trashed Items

**Endpoint**: `GET /api/trash`

Lists soft-deleted items across all collections for the current tenant, sorted by deletion date (newest first). The endpoint queries each collection in sequence and filters for items where `isDeleted === true`.

**Query Parameters**:

| Parameter | Type | Default | Description |
| :-------- | :--- | :------ | :---------- |
| `limit` | `number` | `50` | Maximum number of results **per collection**. Keep this value small for large instances. |

**Response**:

```json
{
  "success": true,
  "data": [
    {
      "_id": "65f...",
      "collectionId": "Posts",
      "collectionName": "Blog Posts",
      "title": "Old Post",
      "deletedAt": "2026-03-18T10:00:00Z",
      "deletedBy": "user_123"
    }
  ]
}
```

### 2. Restore an Item

**POST** `/api/trash/restore`

Restores a specific item from the trash, de-mangling its unique fields and restoring its active status.

**Body**:

```json
{
  "collectionId": "Posts",
  "entryId": "65f..."
}
```

> [!NOTE]
> **Collision Handling**: If restoring an item would cause a unique-index collision (e.g., another active entry now uses the same slug), the API returns `409 Conflict`. The mangled suffix (e.g., `_DELETED_1710793389`) must be manually resolved before retrying.

### 3. Permanent Deletion

To hard-delete an item without sending it to the trash, pass `?permanent=true` on the standard collection delete endpoint.

**Endpoint**: `DELETE /api/collections/[collectionId]/[entryId]?permanent=true`

> [!CAUTION]
> This action is irreversible. The entry and all its revision history will be permanently removed from the database.

---

## 🛡️ Security

- **Tenant Isolation**: Users can only view or restore items within their own tenant.
- **Permissions**: Requires `content:delete` permission for the target collection.
